# Victor AI 面试助手 - 功能需求规格说明书

## 1. 用户体系

### 1.1 用户注册
- **功能描述**：用户通过邮箱/用户名注册账号
- **输入**：用户名、邮箱、密码
- **输出**：注册成功/失败提示
- **业务规则**：
  - 用户名唯一
  - 邮箱格式校验
  - 密码强度要求：至少8位，包含字母和数字

### 1.2 用户登录
- **功能描述**：用户通过账号密码登录系统
- **输入**：用户名/邮箱、密码
- **输出**：登录凭证（Token）
- **业务规则**：
  - 登录失败次数限制（5次锁定15分钟）
  - Token有效期7天

### 1.3 个人中心
- **功能描述**：用户查看和编辑个人信息
- **功能点**：
  - 修改密码
  - 修改昵称/头像
  - 查看使用统计（面试次数、练习时长等）

## 2. 资料系统

### 2.1 题库管理

#### 2.1.1 题目列表
- **功能描述**：展示用户可见的面试题列表
- **筛选条件**：领域、标签、难度、类型（技术/行为）
- **排序方式**：创建时间、更新时间、使用次数

#### 2.1.2 题目详情
- **字段定义**：
  - 题目标题
  - 题目描述
  - 题目类型（技术/行为）
  - 所属领域/标签
  - 难度等级
  - 参考答案（关联原始文档）
  - 关联知识点
  - 来源（系统默认/用户创建/开放接口导入）

#### 2.1.3 题目CRUD
- **创建**：用户手动添加题目
- **编辑**：修改题目内容和答案
- **删除**：软删除，关联数据保留
- **导入**：通过开放导入接口批量导入

### 2.2 岗位库管理

#### 2.2.1 岗位信息
- **字段定义**：
  - 岗位名称
  - 岗位描述/JD原文
  - 技能要求
  - 经验要求
  - 学历要求
  - 薪资范围
  - 所属行业/领域
  - 关联面试题

#### 2.2.2 岗位来源
- 用户手动创建
- 开放接口导入
- 模板一键创建

### 2.3 简历库管理

#### 2.3.1 简历上传
- **支持格式**：PDF、Word、Markdown
- **解析方式**：AI自动解析为结构化数据
- **触发方式**：用户手动触发解析

#### 2.3.2 简历结构化数据
- 基本信息：姓名、联系方式、求职意向
- 教育经历：学校、专业、时间、学位
- 工作经历：公司、职位、时间、职责描述
- 项目经历：项目名称、角色、技术栈、成果
- 技能清单：技能名称、熟练度

#### 2.3.3 简历编辑
- 用户可手动修正AI解析结果
- 用户可手动添加简历内容（无需上传文件）

#### 2.3.4 向量嵌入
- **触发方式**：用户手动触发
- **存储要求**：
  - 每个向量块关联原始文件元数据
  - 文件更新/删除时，向量数据同步更新/删除

### 2.4 经历库管理

#### 2.4.1 经历类型
- 项目经历
- 工作经历
- 教育经历
- 其他经历（证书、获奖等）

#### 2.4.2 经历字段
- 经历类型
- 标题
- 时间范围
- 详细描述
- 关联技能/知识点
- 关联证明材料（图片、文档）

## 3. 面试流程

### 4.1 面试前设置

#### 4.1.1 基础设置（必选）
| 设置项 | 说明 | 必填 |
|-------|------|-----|
| 面试模式 | 语音/文字 | 是 |
| 面试预设 | 选择系统预设配置（不单独持久化模板） | 否 |
| 目标岗位 | 选择岗位库中的岗位 | 是 |
| 关联简历 | 选择简历库中的简历 | 是 |

#### 4.1.2 进阶设置（可选）
| 设置项 | 说明 | 默认值 |
|-------|------|-------|
| 面试轮次 | 配置轮数和每轮重点 | 系统预设 |
| 各维度难度 | 按知识点/技能配置难度 | 自动评估 |
| 时长预设 | 每轮预计时长 | 无限制 |
| 提示功能 | 是否开启提示 | 关闭 |

#### 4.1.3 高级设置（可选）
| 设置项 | 说明 | 默认值 |
|-------|------|-------|
| Agent团队 | 选择或自定义Agent团队 | 系统默认 |
| 模型实例 | 配置各Agent使用的模型 | 系统默认 |
| 追问深度 | 配置追问策略 | 中等 |
| 评分标准 | 自定义评分维度和权重 | 系统默认 |
| 召回策略 | 向量检索/关键词检索/混合检索 | 混合检索 |

### 4.2 召回数据配置

> **触发条件**：基础和高级配置完成后，系统自动进入召回数据配置步骤。

#### 4.2.1 召回配置项
| 设置项 | 说明 | 默认值 |
|-------|------|-------|
| 自动召回 | 是否启用Agent自动召回 | 开启 |
| 召回策略 | 向量检索/关键词检索/混合检索 | 混合检索 |
| 数据来源 | 题库、JD库、资料库、项目经历、简历 | 全选 |
| 每类最大数量 | 每种数据类型的最大召回数量 | 50 |
| 自定义条件 | 用户自定义召回条件 | 无 |

#### 4.2.2 召回数据配置流程
1. Agent根据岗位要求、简历和面试配置自动召回相关资料
2. 前端展示召回结果列表（按数据来源分类）
3. 用户审阅召回结果，可删除不需要的文档
4. 用户可手动添加资料库中的其他文档
5. 用户确认最终召回列表
6. 用户点击创建完成后，Agent基于确认的召回列表异步生成面试题

#### 4.2.3 召回数据来源类型
| 来源类型 | 说明 | 召回依据 |
|---------|------|---------|
| 题库 | 用户创建或导入的面试题 | 岗位技能、面试主题 |
| JD库 | 岗位描述文档 | 关联的目标岗位 |
| 项目经历 | 用户的项目经历 | 简历关联、技能匹配 |
| 简历 | 用户简历 | 关联的简历 |

### 4.3 面试预设

- 系统可提供技术面试、行为面试、综合面试等预设配置。
- 预设配置只用于快速填充轮次、难度、时长等字段，不单独作为数据库模板表持久化。
- 用户可以基于历史面试复制配置，选择重新生成题目或复用已生成题目。

### 4.4 面试执行

#### 4.4.1 面试开始
- 加载面试配置
- 初始化Agent团队
- 准备知识库和资料

#### 4.4.2 面试过程
- AI根据配置出题
- 支持追问机制
- 实时字幕（语音模式）
- 可提交文字/图片补充回答
- 代码编写（Monaco Editor）
- 架构图绘制（Excalidraw）

#### 4.4.3 暂停/恢复
- 用户可随时暂停面试
- 面试状态保存
- 恢复后从断点继续

#### 4.4.4 时长提醒
- 到达预设时长时提醒
- AI询问用户是否继续
- 用户确认后可延长

#### 4.4.5 提示功能
- 用户主动请求提示
- 使用提示扣分（如扣10%该题分数）
- 提示内容从知识库检索

### 4.5 面试结束
- 生成面试记录
- 触发评估流程
- 生成面试报告

## 4. 面试交互

### 5.1 语音模式

#### 5.1.1 语音链路

语音交互流程：用户语音输入 → ASR转文字 → 文字+Prompt发送给LLM → LLM推理返回文字 → TTS转语音 → 播放给用户

#### 5.1.2 语音服务配置
- **TTS服务**：支持多家（千问、火山），用户自配
- **ASR服务**：支持多家（千问、火山），用户自配
- **延迟要求**：尽量低延迟，优化用户体验

#### 5.1.3 实时字幕
- ASR转写的文字实时显示
- AI回复的文字实时显示

#### 5.1.4 补充输入
- 语音模式下可随时切换输入文字
- 支持上传图片（代码截图、架构图等）
- 补充内容与语音回答合并记录

### 5.2 文字模式
- 纯文字对话
- 支持富文本输入
- 支持图片上传

### 5.3 代码编辑
- **组件**：Monaco Editor
- **功能**：语法高亮、代码补全、多语言支持
- **交互**：代码块作为回答的一部分

### 5.4 架构图绘制
- **组件**：Excalidraw
- **功能**：手绘风格图表、流程图、架构图
- **交互**：绘图作为回答的一部分

## 5. 面试评估

### 6.1 评估团队

#### 6.1.1 Agent角色
| Agent角色 | 职责 |
|----------|------|
| 技术评估Agent | 评估技术能力、代码质量 |
| 行为评估Agent | 评估沟通能力、软技能 |
| 领域专家Agent | 评估特定领域知识深度 |
| 综合评估Agent | 汇总各维度评分，生成综合报告 |

#### 6.1.2 Agent配置
- 每个Agent可配置使用的模型实例
- 每个Agent可配置评估维度和权重

### 6.2 难度评估

#### 6.2.1 多维度动态评估
- 结合用户简历、岗位要求、经历综合评估
- 每个维度独立评估难度
- 示例：
  - Java基础：中级（用户有经验）
  - 分布式系统：初级（用户较少接触）
  - 项目管理：高级（用户有丰富经验）

#### 6.2.2 评估输入
- 用户自设基础难度
- 简历解析结果
- 岗位要求
- 用户经历库

### 6.3 追问机制

#### 6.3.1 追问判断
- Agent根据回答质量判断是否需要追问
- 判断依据：
  - 回答深度是否足够
  - 是否涉及关键知识点
  - 岗位要求的核心技能是否覆盖

#### 6.3.2 追问策略
- 深挖细节
- 追问关联知识点
- 场景化提问

### 6.4 参考答案
- **关联原始文档**：答案来源必须有据可依
- **不支持纯AI编造**：无关联文档的题目需标注"无参考答案"

## 9. 开放接入

### 10.1 开放API Key管理
- 用户可以创建开放API Key，用于外部采集程序调用导入接口。
- API Key支持启用/禁用、可选过期时间、权限范围配置。
- 权限范围当前包括：导入岗位、导入题目、导入简历、导入经历。

### 10.2 开放数据导入
- 支持按资料类型分开的导入接口：岗位、题目、简历、经历。
- 外部采集程序如何运行不由系统管理，可以是浏览器扩展、Shell脚本、远程服务或内部程序。
- 导入数据直接写入资料库，并根据Key配置进入正式状态或待审核状态。

### 10.3 待审核资料
- 当API Key的默认导入状态为待审核时，导入资料不会参与AI召回。
- 用户可在资料列表中筛选待审核资料，并执行确认或拒绝。
- 确认后资料进入正式状态，可用于面试召回和题目生成。

### 10.4 来源追踪
- 开放接口导入的资料需要记录来源URL、外部ID、原始导入数据和导入Key。
- 用户查看资料详情时可以看到来源信息，便于排查采集质量。
