## 3. 项目结构

### 3.1 后端项目结构

后端采用分层架构设计，主要包含以下模块：

**核心模块说明：**
- **config/** - 配置类目录，包含安全配置、WebSocket配置、Redis配置等
- **common/** - 公共模块，包含统一响应封装、异常处理、常量定义
- **auth/** - 认证模块，负责用户注册登录和JWT Token管理
- **user/** - 用户模块，管理用户信息和个人中心功能
- **resource/** - 资料模块，管理题库、岗位、简历、经历等资料
- **interview/** - 面试模块，包含面试引擎、WebSocket通信和报告生成
- **agent/** - Agent模块，包含Agent引擎、LLM适配层和工具管理
- **voice/** - 语音模块，提供ASR和TTS服务适配
- **openapi/** - 开放接入模块，管理API Key和数据导入

### 3.2 前端项目结构

前端采用Vue 3 + TypeScript技术栈，主要包含以下模块：

**核心模块说明：**
- **router/** - 路由配置，管理页面导航
- **stores/** - Pinia状态管理，包含用户、面试、资料等状态
- **api/** - API调用封装，按模块组织接口请求
- **views/** - 页面组件，包含认证、资料管理、面试房间、报告中心等页面
- **components/** - 可复用组件，包含通用组件、面试组件（语音录制、实时字幕、代码编辑器、架构图编辑器）和报告组件
- **composables/** - 组合式函数，封装WebSocket、音频处理、面试逻辑等复用逻辑
- **utils/** - 工具函数库
- **types/** - TypeScript类型定义

## 4. 开发环境搭建

### 4.1 前置条件

- JDK 21+
- Node.js 18+
- Docker & Docker Compose
- PostgreSQL 16+ (或使用Docker)
- Redis 7+ (或使用Docker)

### 4.2 后端启动

**启动步骤：**
1. 启动依赖服务：执行 `docker-compose up -d postgres redis` 启动PostgreSQL和Redis
2. 配置环境变量：复制 `.env.example` 为 `.env`，填写数据库密码、JWT密钥等配置
3. 编译项目：执行 `mvn clean package -DskipTests`
4. 启动服务：运行 `java -jar target/victor-server.jar --spring.profiles.active=dev`

### 4.3 前端启动

**启动步骤：**
1. 安装依赖：执行 `npm install`
2. 配置环境变量：复制 `.env.example` 为 `.env`，填写API地址
3. 启动开发服务器：运行 `npm run dev`

### 4.4 Docker Compose启动（全栈）

执行 `docker-compose up -d` 即可一键启动所有服务。

## 5. Docker配置

### 5.1 docker-compose.yml

```yaml
version: '3.8'

services:
  postgres:
    image: pgvector/pgvector:pg16
    environment:
      POSTGRES_DB: ${POSTGRES_DB:-victor_ai}
      POSTGRES_USER: ${POSTGRES_USER:-victor}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-victor123}
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U victor"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    command: redis-server --requirepass ${REDIS_PASSWORD:-redis123}
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  victor-server:
    build:
      context: .
      dockerfile: Dockerfile
    environment:
      DB_HOST: postgres
      DB_PORT: 5432
      DB_NAME: ${POSTGRES_DB:-victor_ai}
      DB_USERNAME: ${POSTGRES_USER:-victor}
      DB_PASSWORD: ${POSTGRES_PASSWORD:-victor123}
      REDIS_HOST: redis
      REDIS_PORT: 6379
      REDIS_PASSWORD: ${REDIS_PASSWORD:-redis123}
      JWT_SECRET: ${JWT_SECRET}
      FILE_STORAGE_PATH: /data/victor/files
    ports:
      - "8080:8080"
    volumes:
      - file_storage:/data/victor/files
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy

  victor-web:
    build:
      context: ./victor-ai-web
      dockerfile: Dockerfile
    ports:
      - "80:80"
    depends_on:
      - victor-server

volumes:
  postgres_data:
  redis_data:
  file_storage:
```

### 5.2 后端Dockerfile

```dockerfile
FROM eclipse-temurin:21-jdk-alpine AS build
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
COPY --from=build /app/target/victor-server.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```
